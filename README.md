# file-organizer-tool
- 指定されたディレクトリ内のファイルを、拡張子に基づいて自動的にフォルダ分けするPythonツールです。

# Features
- 自動フォルダ振り分け: 拡張子ごとにあらかじめ定義されたフォルダへ自動移動します。
- 重複回避（リネーム機能）: 移動先に同名ファイルが存在する場合、(1), (2) のように空き番号が見つかるまで自動で連番を付与し、既存ファイルの上書きを防止します。

- 実行結果の集計表示: 処理完了後、各カテゴリ（フォルダ）ごとに何件のファイルを移動したかのサマリーを表示します。

- 外部設定ファイル（JSON）対応: 振り分けルールを config.json で管理しているため、プログラムを修正せずに拡張子の追加や変更が可能です。

- 例外処理機能: 処理中にファイルがロックされている、あるいはアクセス権限がない場合でも、エラー内容を記録して次のファイル処理を継続します。

- ロギング: 実行時刻、移動の成否、最終的な集計結果を log ファイルへ自動記録します。

- ログのディレクトリ管理: 実行ファイル直下ではなく、専用の Log フォルダ内に実行結果を保存します。

- 日付別ロギング: 実行日ごとに個別のログファイル（例：20260304_sort.log）を生成します。これにより、特定の日の実行履歴を容易に遡ることができます。

- ターゲットディレクトリの動的指定: 整理したい場所を config.json から自由に変更可能。

- ホームディレクトリ対応: ~ 記法をサポート。ユーザー名に依存しない汎用的な設定が可能です。

- 安全なバリデーション: 指定されたパスが存在しない、またはフォルダでない場合は実行前にエラーを表示します。

- シミュレーションモード: config.json の "dry_run": true を設定することで、実際の移動を行わずにシミュレーション結果を表示。

- ディレクトリ回避機能: ターゲット内のフォルダを誤ってスキャンしないよう、ファイルのみを対象にする安全フィルターを搭載。

- 実行サマリー表: 処理終了後、どのフォルダに何件移動したかを、視認性の高いテーブル形式で表示します。

- 合計カウント: 移動した全ファイルの総数を自動集計して表示。

- キーワード除外機能: config.json の exclude_keywords に登録された単語が含まれるファイルは、自動的に仕分け対象から除外されます。(.lowerを使用して大文字と小文字を等しく認識します)

- Python 
- 使用ライブラリ: pathlib.Path, datetime.datetime, json, sys, os

# App Profile
- 開発ツール: VSCode / Wakatime（作業時間の可視化）

# Installation
- リポジトリをクローンし、対象のディレクトリへ移動します。

```bash
git clone [https://github.com/t-seto/file-organizer-tool.git](https://github.com/t-seto/file-organizer-tool.git)
cd file-organizer-tool
```

# Usage
- config.json を編集し、仕分けたいフォルダ名と拡張子のペアを記述します。
- 整理したいファイル群と同じ階層に main.py を配置し、実行します。

```bash
python main.py
```

# Note
- 安全な反復処理: pathlib.iterdir() をリストとしてスナップショット化してからループを回すことで、処理中のファイル移動に伴う FileNotFoundError を回避する設計にしています。

- パス操作の堅牢性: OS依存の文字列結合を避け、pathlib の / 演算子や with_stem() を活用してパスを生成しています。

- ログ管理: ログは追記モードで保存されるため、過去の履歴もすべて保持されます。

- エンコーディング: 日本語のファイル名やフォルダ名に対応するため、UTF-8エンコーディングでログを書き込みます

- ログの永続化: 本ツールは過去のログを自動削除しません。必要に応じて手動でアーカイブ、または削除を行ってください。

- 実行の安全確認: 設定ファイル（config.json）の存在と形式を事前にチェックし、異常がある場合は実行前に処理を停止する安全装置を搭載しています。
